🎛️ CalendarHeader ActionsSection

灵活的自定义操作区域，支持任意内容

✨ 新功能特性

完全自定义

支持任何 React 组件，包括按钮、选择器、输入框、图标等。 完全由开发者控制内容和样式。

预设样式类

提供 primaryButton、secondaryButton、iconButton 等预设样式类， 快速构建一致的用户界面。

响应式设计

在移动端自动调整布局，支持换行和居中对齐， 确保在各种设备上都有良好的用户体验。

灵活布局

支持任意数量的操作项，自动排列和间距调整。 可以包含按钮、选择器、搜索框等各种元素。

📝 使用示例

通过 headerActions 属性传递自定义内容到 CalendarHeader 的 actionsSection 区域。

// 基本使用 <DayView headerActions={ <> <button className="primaryButton">+ Add Event</button> <select> <option value="day">Day</option> <option value="week">Week</option> </select> <button className="iconButton"> <RefreshIcon /> </button> </> } currentDate={currentDate} onDateChange={handleDateChange} />

🎨 预设样式类

// 主要操作按钮 <button className="primaryButton">Primary Action</button> // 次要操作按钮 <button className="secondaryButton">Secondary Action</button> // 图标按钮 <button className="iconButton"> <Icon /> </button> // 危险操作按钮 <button className="dangerButton">Delete</button>

💡 Pro Tip: actionsSection 支持任何 React 内容，包括自定义组件、表单元素、图标等。 使用预设的 CSS 类可以快速构建一致的用户界面，也可以完全自定义样式。
